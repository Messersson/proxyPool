自定义对外接口（独立规则）

怎么填、怎么调（跨设备访问）

完整调用地址 = 协议 + 服务器IP/域名 + 端口 + /open/slug。这里的 slug 只是路径名；真正给其他设备调用时，要把本机/服务器的 IP 和端口写进去。

  • 服务监听：在“配置”页看 HOST/PORT。部署到服务器时通常： HOST=0.0.0.0（监听所有网卡，允许外机访问）， PORT=5010（或你自己选的端口，如 8080/9000）。
  • 其他设备怎么调： 局域网用 http://192.168.x.x:5010/open/crawler-a； 公网/服务器用 http://你的公网IP或域名:端口/open/crawler-a。 不要写 127.0.0.1 给别的机器，那只代表“对方自己”。
  • 防火墙/安全组：服务器上要放行这个端口（Windows 防火墙 / 云厂商安全组）。否则本机能开，外机连不上。
  • 接口标识 slug：路径后缀，只能字母数字 _-，例如 crawler-a。保存后路径是 /open/crawler-a。
  • 名称：仅后台显示，不影响调用。
  • 策略： round_robin=轮流取； lowest_latency=永远最低延迟； random=随机； sticky=同一客户端尽量粘住同一代理（配合租约）。
  • 代理池： auto=优先 HTTP 池，没有再取协议节点； http=只取 HTTP/HTTPS 代理； node=只取协议节点（anytls/vmess/ss 等）。
  • 租约秒数：同一 client_id 在该时间内复用同一代理。默认 300。
  • 优先低延迟 / 跳过超时 / 最大延迟：过滤与优选；最大延迟填 0 表示不限制。
  • 节点类型：仅 node 池生效，如 anytls；留空不限。
  • 指定HTTP代理：一般留空。填了就固定返回它，不再自动选。
  • 接口Token：这个接口自己的密钥。外网建议一定设置。调用时放请求头 X-API-Token，或 query token=。
# 本机自测（仅当前电脑）
GET http://127.0.0.1:5010/open/crawler-a?client_id=job-1
# 其他设备 / 服务器部署后（把 IP 和端口换成实际值）
GET http://服务器IP:5010/open/crawler-a?client_id=job-1
# 带 Token（推荐外网）
curl -H "X-API-Token: 你的token" "http://服务器IP:5010/open/crawler-a?client_id=job-1"
# 强制换下一个代理
GET /open/crawler-a?client_id=job-1&rotate=1
# 返回重点：item.proxy / item.proxy_url / lease_remain_seconds
